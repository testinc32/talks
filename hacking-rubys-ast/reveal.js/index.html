Stage 1: Tokenization

...is the process of breaking a stream of text up into words, phrases, symbols, or other meaningful elements called tokens

			    pry(main)> Ripper.tokenize "def foo; 'hola!'; end"
					=> ["def", " ", "foo", ";", " ", "'", "hola!", "'", ";", " ", "end"]

Stage 2: Lexing

...is the process of giving meaning to these tokens

					pry(main)> Ripper.lex "def foo; 'hola!'; end"
					=> [[[1, 0], :on_kw, "def"],
					 [[1, 3], :on_sp, " "],
					 [[1, 4], :on_ident, "foo"],
					 [[1, 7], :on_semicolon, ";"],
					 [[1, 8], :on_sp, " "],
					 [[1, 9], :on_tstring_beg, "'"],
					 [[1, 10], :on_tstring_content, "hola!"],
					 [[1, 15], :on_tstring_end, "'"],
					 [[1, 16], :on_semicolon, ";"],
					 [[1, 17], :on_sp, " "],
					 [[1, 18], :on_kw, "end"]]

Stage 3: Parsing

...is the process of taking those tokens and creates an abstract syntax tree

Why "abstract"?

# Show AST again

# From the Rubinius docs: Note that the following if expression: rbx compile -S -e ":foo if :bar" [:script, [:if, [:lit, :bar], [:lit, :foo], nil]] and the same if expression but written differently rbx compile -S -e "if :bar then :foo; end" [:script, [:if, [:lit, :bar], [:lit, :foo], nil]] produce exactly the same syntax tree. Because the tree doesn't represent every detail that appears in the real syntax it is called "abstract".

Detect high level code smells

A code smell is a hint that something might be wrong, not a certainty. A perfectly good idiom may be considered a code smell because it’s often misused, or because there’s a simpler alternative that works in most cases. Calling something a code smell is not an attack; it’s simply a sign that a closer look is warranted.
